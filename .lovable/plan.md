## הבעיה
בשדות מותאמים אישית, התווית באנגלית נשמרת נכון במסד הנתונים (בעמודה `labels.en`), אבל כאשר טוענים מחדש את השדות במסך הניהול - הקוד מציב את התווית בעברית גם כתווית האנגלית. ברגע שמשתמש שומר שוב את השדה, התווית בעברית "דורסת" את התווית האנגלית האמיתית גם במסד הנתונים.

## מקור הבאג
ב-`src/pages/Admin.tsx` בארבעה מקומות (שורות 93, 115, 514, 536) הקוד עושה:
```ts
labelEn: field.label,  // ← מציב את העברית כאנגלית
```
במקום לקרוא את האנגלית מתוך `field.labels.en`.

## התיקון
שינוי ארבע השורות לכך שיקראו את האנגלית מ-`labels.en` עם fallback ל-`field.label`:
```ts
const dbLabels = (field.labels as Record<string, string>) || {};
// ...
labelEn: dbLabels.en || field.label,
labels: dbLabels,
```

## קבצים שישתנו
- `src/pages/Admin.tsx` — תיקון 4 המקומות שבהם נטענים שדות מותאמים אישית מהמסד.

## מה לא משתנה
- אין שינוי בסכמת מסד הנתונים.
- אין שינוי בלוגיקת השמירה (היא כבר נכונה: `labels.en = field.labelEn`).
- אין שינוי בלוגיקת הרינדור ב-RSVP/OpenRSVP.

## בדיקה לאחר התיקון
1. ערוך שדה מותאם אישית, וודא שהתווית באנגלית מוצגת נכון בעריכה.
2. שמור ללא שינוי וודא שהאנגלית נשמרת באנגלית במסד.
3. צפה בטופס ה-RSVP בשפה האנגלית — התווית צריכה להיות באנגלית.
